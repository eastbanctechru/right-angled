General idea

The directives from this section work directly with the native HTML elements and browser events. They implement useful features not related directly to the right-angled library. For example, you can use rtStopEvents directive for preventing event bubbling on any element.

How to bootstrap

To work with these directives you should import the RTMiscModule module from right-angled library and add it into your AppModule imports array. You can also use RTModule module which includes the functionality of the entire library.

rtStopEvents directive

This directive gets a string or array of string with the event names as a parameter.

The directive will subscribe to the specified events of the element and will call the event stopPropagation method in the event handler to prevent further event propagation.

In the example below you can see, how the input element, marked by this directive, stops the keypress event propagation if the "stop keypress events propagation" checkbox is checked and does not allow the parent div element to process this event. If you uncheck "stop keypress events propagation" checkbox, keypress event handler of the div element begins to log every key that has been pressed.

rtPreventDefaults directive

This directive gets a string or array of string with the event names as a parameter.

The directive will subscribe to the specified events of the element and will call the event preventDefault method in the handler, preventing performing the default actions for the event.

rtSelectOnFocus directive

This directive is used with the input elements and selects all input text when element gets focused.

rtFocusOnRender directive

This directive is some kind of replacement for HTML autofocus attribute, but it works with the dynamic rendering.

The input element marked by this directive will be focused automatically right after being rendered.
